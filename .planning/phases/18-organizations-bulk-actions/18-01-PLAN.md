  <action>
Add bulk update endpoint to RONDO_REST_Teams class, following the pattern from class-rest-people.php:

1. In register_routes(), add route registration for POST `/rondo/v1/teams/bulk-update`:
   - Permission callback: check_bulk_update_permission
   - Args: ids (required array), updates (required object with visibility, assigned_workspaces, labels_add, labels_remove)

2. Add check_bulk_update_permission() method:
   - Verify user is logged in
   - Loop through all IDs, verify each post exists and is type 'team'
   - Verify current user is post author OR administrator
   - Return WP_Error for invalid/forbidden IDs, true on success

3. Add bulk_update_teams() callback method:
   - Process each post ID in try/catch
   - If visibility provided: use RONDO_Visibility::set_visibility()
   - If assigned_workspaces provided: convert post IDs to term IDs using 'workspace-' slug pattern, call wp_set_object_terms() and update_field('_assigned_workspaces')
   - If labels_add provided: wp_set_object_terms($post_id, $term_ids, 'team_label', true) (append mode)
   - If labels_remove provided: wp_remove_object_terms($post_id, $term_ids, 'team_label')
   - Return response with success boolean, updated array, failed array

Note: Teams do NOT have team_id update (unlike People) - they ARE teams. Only visibility, workspaces, and labels.


2. In src/hooks/useTeams.js, add useBulkUpdateTeams hook following useBulkUpdatePeople pattern:
   ```javascript
   export function useBulkUpdateTeams() {
     const queryClient = useQueryClient();

     return useMutation({
       mutationFn: async ({ ids, updates }) => {
         const response = await wpApi.bulkUpdateTeams(ids, updates);
         return response.data;
       },
       onSuccess: () => {
         queryClient.invalidateQueries({ queryKey: ['teams'] });
       },
     });
   }
   ```
