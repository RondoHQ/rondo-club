# Phase 140: Frontend Messaging - Context

**Gathered:** 2026-02-04
**Status:** Ready for planning

<domain>
## Phase Boundary

Add UI messaging to make clear that tasks are personal. Users should understand their tasks are only visible to them. No new capabilities — just informational text in strategic locations.

</domain>

<decisions>
## Implementation Decisions

### Note placement
- Tasks list page: Note in page header area, always visible (not dismissible)
- Task creation modal: Note at top of modal, before form fields
- Both locations show the personal visibility message

### Message tone
- Language: Dutch (matches rest of UI)
- Tone: Informational, neutral, factual
- Tasks list page: "Taken zijn alleen zichtbaar voor jou"
- Task creation modal: "Deze taak is alleen zichtbaar voor jou"

### Claude's Discretion
- Visual treatment (info box style, icon choice, colors)
- Exact positioning within header/modal
- Whether to include an icon alongside the text

</decisions>
