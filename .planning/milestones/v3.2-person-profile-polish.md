# Milestone v3.2: Person Profile Polish

**Status:** SHIPPED 2026-01-14
**Phases:** 29-31
**Total Plans:** 3

## Overview

Enhance the PersonDetail page with role/job display in header, a persistent todos sidebar visible across all tabs, and mobile-responsive todo access via floating action button with slide-up panel.

## Phases

### Phase 29: Header Enhancement

**Goal**: Add role/job display to person header showing current position and organization
**Depends on**: Previous milestone complete
**Plans**: 1 plan

Plans:
- [x] 29-01: Add role/job display to person header

**Details:**
- Added `currentPositions` useMemo to filter jobs where `is_current` is true
- Display current position below name, before nickname/demographics
- Format: "Job Title at Company" with Company as clickable link
- Handles edge cases: multiple positions (comma-separated), title-only, company-only

### Phase 30: Todos Sidebar

**Goal**: Create persistent sidebar showing person's todos across all tabs
**Depends on**: Phase 29
**Plans**: 1 plan

Plans:
- [x] 30-01: Persistent todos sidebar

**Details:**
- Created two-column flex layout for tab content area
- Moved todos from Timeline tab to persistent sidebar
- Added sticky positioning for sidebar scroll behavior
- Added open todos count badge next to header
- Hidden on screens below lg breakpoint

### Phase 31: Person Image Polish

**Goal**: Responsive behavior, collapse/expand, mobile handling
**Depends on**: Phase 30
**Plans**: 1 plan

Plans:
- [x] 31-01: Mobile todos access (FAB + slide-up panel)

**Details:**
- Added floating action button visible on mobile (< lg breakpoint) at bottom-right
- FAB shows open todos count badge when > 0
- Slide-up panel with backdrop, drag indicator, and header controls
- Full todo functionality in mobile panel (toggle, edit, delete, add)
- Added CSS animation for smooth panel slide-up
- Fixed timeline endpoint to support new todo post statuses
- Changed to 3-column grid layout for equal-width columns

## Milestone Summary

**Key Decisions:**
- Current positions derived from existing sortedWorkHistory using useMemo
- "Works at Company" fallback when job title missing
- Sidebar hidden below lg breakpoint (mobile gets FAB instead)
- Badge counts open + awaiting todos (excludes completed)
- FAB at z-40 (above content, below modals)
- Panel closes on edit/add actions before modal opens
- 3-column grid layout for equal column widths

**Issues Resolved:**
- Timeline endpoint updated for todo post statuses (was using deprecated 'publish' status and 'is_completed' field)

**Issues Deferred:**
- None

- None
