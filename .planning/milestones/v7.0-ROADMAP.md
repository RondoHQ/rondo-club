# Milestone v7.0: Dutch Localization

**Status:** SHIPPED 2026-01-25
**Phases:** 99-106
**Total Plans:** 22

## Overview

This milestone transforms the entire Stadion frontend from English to Dutch. Starting with date formatting foundation (Dutch locale for date-fns), then systematically translating each UI area: navigation, dashboard, entity pages (Leden, Teams, Commissies), important dates, todos, settings, and global UI elements. Direct string replacement approach - no i18n library complexity.

## Milestone Summary

**Key Decisions:**

- Dutch locale via dateFormat.js wrapper - single import for all date formatting
- Loan words kept (Dashboard, Workspaces, Feedback) - common English terms in Dutch context
- Informal tone (je/jij) - warm, friendly UI
- "Leden" terminology (not Personen) - consistent member/contact naming
- "Openstaand" for awaiting (not Wachtend) - consistent in dashboard and todos
- Mixed terminology for activity types - keep international terms (Email, Chat, Lunch)

**Issues Resolved:**

- Fixed corrupted variable names from partial earlier translations in Settings
- Escaped quotes in empty state text to pass ESLint react/no-unescaped-entities
- Removed unused date function imports after migration to dateFormat.js

**Technical Debt Incurred:**

None - all phases completed with clean verifications.
